     2.1 Мета роботи
     Метою практичного заняття є ознайомитися з одним із шаблонів проєктування програмного забезпечення, проаналізувати його структуру, призначення та практичне застосування.
     2.2 Завдання
     Підготувати доповідь на тему «Шаблон (патерн) проєктування ПЗ: Observer (Спостерігач)», ознайомитися з його структурою, призначенням та практичним застосуванням, розробити, з прикладом коду на C# та сценаріями використання, завантажити програмний код прикладу патерну та експортувати звіт про виконання роботи у встановленому форматі.


3 ОПИС ВИКОНАНОЇ РОБОТИ
     У процесі розробки програмного забезпечення часто виникає необхідність організувати взаємодію між об’єктами таким чином, щоб зменшити їхню залежність один від одного та підвищити гнучкість системи. Саме для цього використовуються шаблони проєктування.
     Шаблони (патерни) проєктування - це перевірені часом типові рішення поширених проблем у проєктуванні програмного забезпечення. Вони не є готовим кодом, а радше описують підхід або структуру, яку можна застосувати в різних ситуаціях. Патерни допомагають розробникам створювати більш гнучкі, масштабовані та зрозумілі системи.
     Патерн Observer сформувався в процесі розвитку об’єктно-орієнтованого програмування як відповідь на необхідність організації взаємодії між об’єктами зі зменшеною зв’язаністю. Його концептуальні основи були закладені ще у 1970–1980-х роках під час розробки інтерактивних систем, зокрема графічних інтерфейсів користувача, де виникла потреба автоматичного оновлення представлення даних при зміні їхнього стану. Формальне визначення та узагальнення патерну було здійснено у 1994 році в роботі Еріхa Гамми, Річарда Гелма, Ральфа Джонсона та Джона Вліссіда «Design Patterns: Elements of Reusable Object-Oriented Software».
     Основна ідея патерну полягає в тому, щоб розділити об’єкт, який генерує події, та об’єкти, які на них реагують, без жорсткого зв’язку між ними. Це дозволяє додавати або видаляти спостерігачів без зміни коду суб’єкта.
     Патерн використовується у: графічних інтерфейсах при оновленні UI при зміні даних, системах підписок,  архітектури MVC або MVVM, системах моніторингу та логування, event-driven системах.
     Важливість патерну: зменшення зв’язаності, підтримка принципу Open-Closed, дозволяє легко масштабувати систему.
     Патерн Observer має чітко визначену структуру, що складається з чотирьох основних компонентів:
   1. Subject (Суб’єкт) - це об’єкт, який зберігає список спостерігачів та надає методи для:
• підписки Attach, Subscribe
• відписки Detach, Unsubscribe
• сповіщення Notify
   2. Observer (Спостерігач) - інтерфейс або абстрактний клас, що визначає метод:
• Update() - викликається при зміні стану суб’єкта
3. ConcreteSubject (Конкретний суб’єкт) - це реалізація Subject, яка:
• зберігає внутрішній стан
• викликає Notify при його зміні
4. ConcreteObserver (Конкретний спостерігач) - Реалізація Observer, яка:
• отримує оновлення
• змінює свій стан відповідно до даних
Розглянемо реалізацію патерну Observer мовою C#: 
Визначення інтерфейсу підписника
Див. додаток B.1 4-7 рядок. На цьому етапі ми створюємо контракт взаємодії між сервісом і всіма підписниками. Інтерфейс визначає єдиний метод Update, через який підписники отримують дані про зміну стану. Це і є основа слабкої зв’язаності: сервіс працює не з конкретними класами, а з абстракцією.
Реалізація конкретних підписників
Див. додаток B.1 9-31 рядок. Тут ми створюю конкретні реалізації спостерігачів, які підписуються на події.
Реалізація сервісу
Див. додаток B.1 33-70 рядок. Це центральний елемент - видавець подій. Сервіс зберігає всіх, хто підписався на оновлення. Він зберігає інтерфейси, а не конкретні класи. Методи Subscribe та Unsubscribe дозволяють динамічно додавати підписників або видаляти їх, це означає, що система може змінюватися під час виконання, а не тільки на етапі компіляції. Метод UpdateRate - це точка, де відбувається зміна стану системи, де після зміни стану викликається NotifySubscribers. Метод NotifySubscribers - це ядро патерну Observer, який проходить по всіх підписниках та викликає у кожного метод Update.


4 ВИСНОВКИ
     Патерн Observer є важливим інструментом у проєктуванні сучасних програмних систем, особливо тих, що базуються на подіях або потребують синхронізації стану між компонентами. Його використання дозволяє створювати гнучкі, масштабовані та легко підтримувані системи, що є критично важливим у сучасній розробці програмного забезпечення.

 2using System.Collections.Generic;
 3
 4public interface ICurrencySubscriber
 5{
 6    void Update(string currency, decimal rate);
 7}
 8
 9public class MobileAppClient : ICurrencySubscriber
10{
11    public void Update(string currency, decimal rate)
12    {
13        Console.WriteLine($"[Mobile] New currency {currency}: {rate}");
14    }
15}
16
17public class WebDashboard : ICurrencySubscriber
18{
19    public void Update(string currency, decimal rate)
20    {
21        Console.WriteLine($"[Web] Updated chart for {currency}: {rate}");
22    }
23}
24
25public class AnalyticsService : ICurrencySubscriber
26{
27    public void Update(string currency, decimal rate)
28    {
29        Console.WriteLine($"[Analytics] Currency display {currency}: {rate} added to DB");
30    }
31}
32
33public class CurrencyRateService
34{
35    private List<ICurrencySubscriber> _subscribers = new();
36
37    public void Subscribe(ICurrencySubscriber subscriber)
38    {
39        _subscribers.Add(subscriber);
40    }
41
42    public void Unsubscribe(ICurrencySubscriber subscriber)
43    {
44        _subscribers.Remove(subscriber);
45    }
46
47    public void UpdateRate(string currency, decimal newRate)
48    {
49        Console.WriteLine($"\n[Service] Currency {currency} updated to {newRate}");
50        NotifySubscribers(currency, newRate);
51    }
52
53    private void NotifySubscribers(string currency, decimal rate)
54    {
55        foreach (var subscriber in _subscribers)
56        {
57            subscriber.Update(currency, rate);
58        }
59    }
60}
61
62public class Program
63{
64    public static void Main(string[] args)
65    {
66        var rateService = new CurrencyRateService();
67
68        var mobile = new MobileAppClient();
69        var web = new WebDashboard();
70        var analytics = new AnalyticsService();
71        
72        rateService.Subscribe(mobile);
73        rateService.Subscribe(web);
74        rateService.Subscribe(analytics);
75        
76        rateService.UpdateRate("USD", 41.25m);
77    }
78}
